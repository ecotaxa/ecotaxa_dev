Taxonomy filter Help.
% allows you to search multiple ordered part on the taxon name
ie: co%pe%da will search something starting by 'co' then containing 'pe' then containing 'da'

Starting by % allows to avoid the implicit rule starting by
ie : %poda search any name containing 'poda' (take care it's slower)

* allows to put criteria on ancestor
ie : %rus*cop*cal will search something starting by 'cal' with an ancestor stating by 'cop' and another ancestor containing 'rus'
Note : criteria before the last * are not ordered, and search only on the first 6 ancestors.

Note : Values underlined in the result list comes from project preset.
